Metrics Guide

1. Task Resolution Rate (TRR)

에이전트가 생성한 patch로 실제 버그가 해결된 비율입니다.

TRR = (FAIL_TO_PASS 테스트를 모두 통과한 태스크 수) / (전체 태스크 수)

Docker 컨테이너 안에서 patch를 적용한 후 FAIL_TO_PASS 테스트를 실행하여 판정합니다. 원래 실패하던 테스트가 모두 통과하면 해당 태스크는 "resolved"로 간주됩니다.

SABCDF
≥ 60%≥ 45%≥ 30%≥ 20%≥ 10%< 10%

2. Regression Safety

patch 적용 후 기존에 통과하던 테스트가 여전히 통과하는 비율입니다.

Regression Safety = (PASS_TO_PASS 테스트가 전부 통과한 태스크 수) / (전체 태스크 수)

버그를 수정하면서 다른 기능을 깨뜨리지 않았는지 확인합니다.

SABCDF
≥ 98%≥ 95%≥ 90%≥ 85%≥ 75%< 75%

3. Token Efficiency

해결된 태스크당 평균 토큰 사용량입니다. 낮을수록 좋습니다.

Token Efficiency = sum(해결된 태스크의 total_tokens) / (해결된 태스크 수)

total_tokens는 input_tokens(cache read/write 포함) + output_tokens입니다. 해결되지 않은 태스크는 계산에서 제외됩니다.

SABCDF
≤ 50K≤ 100K≤ 200K≤ 400K≤ 800K> 800K

4. Cost per Resolved Task (CRT)

해결된 태스크당 평균 API 비용입니다. 낮을수록 좋습니다.

CRT = (전체 태스크의 총 비용) / (해결된 태스크 수)

실패한 시도에 들어간 비용도 포함합니다.

SABCDF
≤ $0.50≤ $1.00≤ $2.00≤ $5.00≤ $10> $10

5. E2E Completion Time

태스크당 평균 종단간 실행 시간(초)입니다. 낮을수록 좋습니다.

E2E Time = avg(task_end - task_start)

에이전트가 문제를 풀고 코드를 수정하는 데 걸린 순수 시간입니다. 레포 클론 시간은 포함되지 않습니다.

SABCDF
≤ 60s≤ 120s≤ 300s≤ 600s≤ 1200s> 1200s

6. Time to First Action (TTFA)

에이전트가 첫 번째 동작을 수행하기까지 걸린 시간(초)입니다. 낮을수록 좋습니다.

TTFA = first_action - task_start

프롬프트를 받은 후 첫 번째 도구 호출까지의 지연 시간입니다.

SABCDF
≤ 3s≤ 5s≤ 10s≤ 20s≤ 30s> 30s

7. Convergence Steps

태스크를 완료하기까지 수행한 평균 턴(스텝) 수입니다. 낮을수록 좋습니다.

Convergence Steps = avg(num_turns)

한 턴은 모델을 한 번 호출하여 응답을 받고 도구를 실행하는 단위입니다. 적은 턴으로 문제를 해결할수록 효율적입니다.

SABCDF
≤ 5≤ 10≤ 20≤ 30≤ 50> 50
